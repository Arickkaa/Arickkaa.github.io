上面的图片看起来十分荒诞，所以要理解操作系统。

理解操作系统

  • “精确”的定义毫无意义，操作系统是一门人造的形式科学，应该问：操作系统是如何发展成现在这样的？

  • 三个重要的线索：

    • 计算机（hardware）
    • 程序（software）
    • 操作系统（OS）

这里只讨论单一的计算机中支撑程序执行的软件系统。

操作系统的发展历程

1. 1940-1950年代

ENIAC（1946.2.14），这一阶段的计算机是一个跨时代的设计，但是相较于今天的计算机来说是非常简单的。

这一时期的计算机属于手工控制阶段，由手工操作进行编程和控制，主要用于管理输入输出设备和控制计算机，这一时期没有操作系统，能让程序在计算机中跑起来就很了不起。

  • 逻辑门：真空电子管
  • 存储器：延迟线
  • 输入输出：打孔纸带/指示灯

2. 1950-1960年代

这一时期开始使用更快更小的逻辑门（晶体管）、更大的内存（磁芯）、多种I/O设备。此时I/O设备的速度严重低于处理器速度，所以开始出现中断机制。

这时的计算机已经可以执行更复杂的任务，包括通用的计算。所以希望使用计算机的人越来越多，大家希望调用API而不是直接访问设备，因此编程语言Fortran（1957）诞生。

但是在当时写程序、跑程序在当时都是非常麻烦的事情。即使是有计算机的学校一个学校通常只有一台计算机（太贵了），但是有很多用户需要使用它，因此产生了集中管理计算机的需求：多用户排队共享计算机。

操作系统的概念开始形成：

  • 操作（operate）任务（jobs）的系统（system）
    • “批处理系统” = 程序的自动切换（换卡） + 库函数API
    • Disk Operating System（DOS）
      • 操作系统中开始出现“设备”、“文件”、“任务”等对象和API

3. 1960-1970年代

出现了集成电路、总线

  • 更快的处理器
  • 更快、更大的内存
    • 可以同时载入多个程序不需要“换卡”
  • 更丰富的I/O设备，完善的中断机制/异常机制
  • 更多高级的语言和编译器出现

这时候的操作系统可以载入多个程序到内存中，并且灵活调度它们的管理程序，包括程序可以调用的API。

同时将多个程序载入内存是一项巨大的能力

  • 出现了进程（process）的概念
  • 在执行I/O时，可以将CPU让给另外的进程
    • 在多个地址空间隔离的程序之间切换
    • 虚拟存储使一个程序出bug不会让整个系统崩溃

操作系统中自然地增加进程管理API

既然操作系统已经可以在程序之间切换，为什么不让它们定时切换呢？

因此中断机制出现：

  • 时钟中断：使程序在执行时，异步地插入函数调用
  • 由操作系荣（调度策略）决定是否要切换到另一个程序执行
  • Multics（MIT，1965），现代操作系统诞生

4. 1970-1980年代

集成电路空前发展，兴起个人计算机（PC），“计算机”已经与今天的没有很大差别

  • CISC指令集；中断、I/O、异常、MMU、网络
  • 个人计算机、超级计算机......

当时出现PASCAL（1970）， C（1972），......高级编程语言。今天能做到的，那个时代大多也已经能办到--图像声音视频、人工智能......同时个人开发者（Geek Network）走上舞台。

基于此背景，分时操作系统走向成熟，UNIX诞生并走向完善，奠定了现代操作系荣的形态。

  • 1973：信号API、管道（对象）、grep（应用程序）
  • 1983：BSD socket（对象）
  • 1984：procfs（对象）
  • UNIX衍生出大家族
    • 1BSD (1977), GNU (1983), MacOS (1984), AIX (1986), Minix (1987), Windows (1985), Linux 0.01 (1991), Windows NT (1993), Debian (1996), Windows XP (2002), Ubuntu (2004), iOS (2007), Android (2008), Windows 10 (2015), ……

5.今天的操作系统

通过“虚拟化”硬件资源为程序运行提供服务的软件。

空前复杂：

  • 更复杂的处理器和内存
    • 非对称多处理器（ARM big.LITTLE；Intel P/E-cores）
    • Non-uniform Memory Access（NUMA）
    • 更多的硬件机制...
  • 更多的设备和资源
    • 网卡、SSD、GPU、FPGA......
  • 复杂的应用需求和环境
    • 服务器、个人电脑、智能手机、手表、手环......
