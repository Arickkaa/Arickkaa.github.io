Q1: Compare the two different methods for declaring functions in Python, using the def and lambda keywords. When would you choose a def function? When would you choose a lambda function? When can you not use a lambda?

Lambda in python is usually used to create anonymous functions in python, while methods created with def have names. If we need to assign the lambda to a name, use a def instead. If we still need some lambda not covered, we might consider writing a def, just to be more readable. If the function is more complex than the ones at operator module, a def is probably better.

Q2: What is variable scope? What is a locally-scoped variable? What is a globally-scoped variable?

The scope of a variable is the part of the program in which it is visible. If we define a variable
as part of a compound statement it is also local to that part of the program.

Variables can be declared inside a function or outside of a function. When declared inside a
function they are known as ‘local variables’ and are only available inside the function itself.

If we declare a variable outside a function it is sometimes called a ‘global scope’ variable. These types of variables can be used (and changed) by code in any function. Variables declared inside one function are not visible to other functions.

Q3: What is a better alternative to using globally-scoped variablesa

Q4: What is special about a function closure and why is it called a closure?

A function closure means that function is passed as an argument and function is passed as return value.

It is called a closure because the function can read internal variables of other functions.

Q5: Describe the necessary parts of a recursive function and what it means for a function to be recursive

A recursive function is a function that calls itself. A recursive computation solves a problem by using the solution of the same problem with simpler inputs. For a recursion to terminate, there must be special cases for the simplest inputs.

Q7: How does a Python program become executable when passed to the Python interpreter? What is the conventional way to control this behavior?

When we run a Python program, Python closure will perform two steps. Compile source code into bytecode and then Forward the compiled bytecode to the Python virtual machine (PVM) for execution.

The conventional way to control this behavior is to compile the source code into binary code and then perform the steps above.

Q10: Describe the function “rest” arguments feature in Python. When is it useful to use rest arguments? When should you not necessarily use rest arguments?

“Rest” arguments are used to obtain redundant parameters of the function, so that the arguments object is not required.
